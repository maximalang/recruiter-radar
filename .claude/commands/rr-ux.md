# rr-ux

Review UX and conversion aspects of current changes. Use for UI components and activation flows.

## Focus areas

1. **Activation flow** — supports core loop?
2. **Quality signals** — evidence visible, confidence levels displayed
3. **Russian copy** — concise, premium, no exaggeration
4. **Telegram digest** — all 8 feedback buttons present, safe next action shown
5. **Accessibility** — keyboard nav, screen reader, error/loading/empty states
6. **No AI aesthetic** — no purple gradients, rounded-2xl, generic layouts

## Output

```
=== UX REVIEW ===
<area>: [pass|fail|partial] <comment>
OVERALL: <pass|needs-work>
```

## When to run

- Building/modifying UI components
- Before completing UX-focused tasks
